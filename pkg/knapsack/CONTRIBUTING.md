# Contributing

Open an issue before changing public packing semantics. Add a focused failing
test, implement the smallest correct change, and run `make check`. Geometry,
verification, pruning, objective, status, and accounting changes also require
mutation coverage and adversarial tests. New reference fixtures need pinned
revision, license, checksum, normalization, and provenance evidence.

Commits use Conventional Commits with an explanatory body. Do not claim
optimality or comparative performance without reproducible proof.
