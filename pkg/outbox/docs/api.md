# API Reference

The authoritative symbol-level reference is generated by `go doc`. This page
summarizes ownership and side effects that are easy to misuse.

## Envelope

- `NewEnvelopeBuilder`: validates injected clock, ID generator, and complete
  size limits.
- `EnvelopeBuilder.Build`: copies payload and metadata, assigns database-scale
  UTC timestamps, and defaults payload version to 1.
- `Envelope.ValidateForInsert`: checks direct envelope construction against
  complete positive limits and new-record persistence invariants.
- `Envelope.CanonicalJSON`: deterministic JSON with stable struct order and
  sorted metadata keys. Explicit timestamp formatting prevents silent
  `time.Time` marshaling failures; insertion accepts JSON-supported years.

## PostgreSQL writer

- `NewWriter`: selects a quoted schema/table, envelope limits, and insert batch
  bound, defaulting to `public.outbox_messages`, `DefaultLimits`, and 100.
- `Insert` and `InsertBatch`: execute through the supplied `pgx.Tx`; batch
  insertion is one statement. Every direct envelope and the complete batch are
  validated before the transaction is touched. Neither method commits. The
  caller must roll back the complete transaction after any writer error.
- `WriterConfig.Limits`: may apply stricter or broader application validation,
  but actual values must still fit the embedded schema ceilings. PostgreSQL
  `jsonb`-incompatible NUL metadata fails with `ErrValueOutsideBounds` before
  SQL execution. The schema also rejects direct-SQL metadata whose top-level
  value is not an object or whose values are not strings.

## PostgreSQL store

- `Claim`: leases a bounded batch using the PostgreSQL clock and returns the
  token needed for every following mutation. Attempts increment on claim and
  saturate at the schema/relay ceiling of 10,000.
- `ExtendLease`, `MarkDelivered`, `Retry`, `DeadLetter`, `ReleaseLease`: mutate
  only a still-leased record with the matching token. Oversized IDs and tokens
  fail with `ErrValueOutsideBounds` before SQL execution.
- `Retry`: accepts a zero-to-one-minute duration, not an absolute timestamp;
  PostgreSQL adds it to `clock_timestamp()`. Invalid direct-call delays return
  `ErrInvalidRetryDelay`.
- `Retry` and `DeadLetter` persist only the bounded diagnostic
  `operation failed`; arbitrary publisher error text is available to the
  in-process classifier but is never copied into PostgreSQL.
- `Replay`: atomically resets a complete terminal selection and writes audit
  records only after `StoreConfig.ReplayAuthorizer` approves the complete
  request. Stores deny replay by default with `ErrReplayUnauthorized`. The
  hook receives a copied ID slice; denial and panic details are discarded.
  Replay intentionally permits duplicate publication. Selection IDs,
  requester, reason, and an explicit schedule's year are validated before
  authorization. The generated audit ID is validated after approval and before
  a transaction begins.
- `PruneDelivered`: deletes a bounded batch of old delivered records only.
- `PruneDead`: deletes a bounded batch of old dead letters only.
- `ArchiveAndPruneDelivered`: locks a bounded delivered batch, passes complete
  immutable messages to an `Archiver`, and deletes them only after the hook
  succeeds. Archives must deduplicate by envelope ID because commit ambiguity
  can invoke archival again on a later attempt.
- `ArchiveAndPruneDead`: applies the same archive-before-delete contract to
  dead letters and includes terminal time and last error in the restricted
  archive record.
- Replay and archive own their PostgreSQL transactions. Deferred rollback uses
  a detached five-second deadline so cancellation still releases resources
  without an unbounded network wait.
- `ErrArchiverPanic`: a payload-safe sentinel returned when an archive hook
  panics; the transaction rolls back and preserves every selected row.
- `Ping`: verifies a PostgreSQL round trip and rejects read-only sessions or
  servers in recovery with `ErrNotWritable`.
- `Backlog`: returns pending, leased, and dead counts together with the oldest
  pending availability time for low-frequency operational checks.
- `Inspect`: returns a bounded, filterable administrative selection containing
  state and timing summaries but never payload or metadata.
- `StoreConfig.Observer`: emits payload-safe replay, prune, and archive outcome
  counts and latency through the same root `Observer` contract as the relay.
- `StoreConfig.Clock`: controls diagnostic durations only. Panics are contained
  and yield zero-time diagnostics without changing database behavior.
- `StoreConfig` enforces absolute ceilings of 1,000 claim/admin records and a
  24-hour lease, even when applications configure broader values.

## Relay

- `New`: validates bounded worker, batch, lease, renewal, retry, polling,
  cleanup, and ordering configuration.
- `RunOnce`: claims and drains one bounded batch.
- `Run`: polls until cancellation and immediately repolls full batches.
- `Readiness`: checks writable-primary PostgreSQL status and, when the
  publisher implements
  `Health(context.Context) error`, publisher connectivity. It joins failures
  so callers can inspect every unavailable dependency.
- `Publisher`: one context-aware method accepting a complete envelope.
- `ErrPublisherPanic`: a payload-safe sentinel used when relay containment
  converts a publisher panic into the normal classification/retry path.
- `ErrHeartbeatPanic`: a payload-safe sentinel returned after a custom
  heartbeat panics. Publication is canceled and the existing lease is left
  untouched for expiry-based recovery.
- `ErrClassifierPanic`, `ErrInvalidErrorClass`, `ErrBackoffPanic`, and
  `ErrInvalidBackoff`: payload-safe policy failures returned after the relay
  durably schedules a bounded transient retry. Custom retry delays are clamped
  to the package's one-minute maximum.
- `ErrClaimBatchOverflow`: a Store violated its contract by returning more
  claims than requested. The relay reports the claimed count and does not
  allocate or publish the oversized response; leases recover by expiry.
- `Config.Heartbeat`: injectable lease-maintenance loop for deterministic
  testing. The default extends each in-flight lease halfway through its
  duration until publication finishes or the relay context is canceled. A
  panic is contained at the relay-owned goroutine boundary.
- `Config.Clock`: controls relay diagnostic durations only. Panics are
  contained and yield zero-time diagnostics without changing relay state.

Relay configuration has absolute ceilings of 1,000 records per batch, 256
workers, 10,000 attempts, and a 24-hour lease. Normal defaults remain 100
records, `runtime.NumCPU()` workers, 10 attempts, and a 30-second lease.

## Observability

- `Observer` receives structured claim, publish, delivery, retry, dead-letter,
  and release events without payloads, metadata, or error text.
- `BacklogStats` is the dependency-free snapshot shared by PostgreSQL and
  optional telemetry integrations.
- `ObserverFunc` adapts a function to `Observer`.
- `relay.Config.Logger` emits the same payload-safe event fields through
  `log/slog`. Applications control handlers, sampling, and export.

Observer and logger panics are contained at the diagnostic callback boundary.
They cannot change a store result or interrupt relay delivery; operators must
monitor their telemetry pipeline independently because the panic is discarded.

Message IDs, topics, and attempts are included for diagnostics. Operators
should apply their own cardinality and retention policies when translating
events into metrics.

Returned operational errors wrap their underlying causes for `errors.Is` and
`errors.As` inspection. Applications must keep those errors out of untrusted
logs because the persistence redaction boundary does not sanitize application
logging.
