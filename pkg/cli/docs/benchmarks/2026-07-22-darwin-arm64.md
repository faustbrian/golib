## Equivalent comparison

The comparison measures construction separately from prepared dispatch. Every
dispatch uses the same `deploy --force target` tree, typed validation, JSON
encoding to `io.Discard`, and successful result. Construction results do not
include dispatch. Dispatch results do not hide construction on only one side:
each implementation first prepares the equivalent graph outside the timer.

| Framework | Construction median | Prepared dispatch median | Dispatch allocations |
| --- | ---: | ---: | ---: |
| `cli` | 1.489 µs | 3.251 µs | 82 |
| Cobra v1.10.2 | 360.9 ns | 1.222 µs | 24 |
| `urfave/cli` v3.5.0 | 211.8 ns | 7.384 µs | 178 |
| Kong v1.16.0 | 6.120 µs | 21.98 µs | 136 |
| standard `flag` | 109.1 ns | 107.9 ns | 1 |

The dispatch samples for `urfave/cli`, Kong, and standard `flag` were noisy on
this short audit run (95% median intervals of 77%, 48%, and 70%). The raw data
and `benchstat` report retain that uncertainty; no cross-framework speed claim
is drawn from it. `cli` was within the release budget of four times prepared
Cobra latency and no more than 100 allocations.

Standard `flag` is a parsing floor and manually dispatches the subcommand
because it has no command tree. It is not feature-equivalent. Prepared
`cli` dispatch creates fresh internal parser state, while direct Cobra
reuses its mutable prepared graph. The difference is the measured cost of
repeated and concurrent invocation isolation.

## Owned scenarios

The root data separately measures small, broad, deep, and maximum graph
construction; root and deep dispatch; typed conversion; usage and validation
failures; suggestions; help; completion; manifests; JSON output; and
cancellation. For orientation, medians included 8.283 µs for a small compile,
1.222 ms for a broad compile, 3.365 ms for a maximum-command compile,
27.17 µs for root dispatch, and 432.2 ns for already-cancelled execution.

## Binary-size measurement

The trimmed reference process fixture was built with:

```sh
GOWORK=off go build -trimpath -o /tmp/cli-process-fixture \
  ./cmd/process-fixture
```

The binary was 4,658,706 bytes. `go tool nm -size` attributed 79,056 bytes to
symbols containing the `cli` module import path. This attribution omits
shared runtime, standard-library, Cobra, and linker effects, so it is an
auditable lower bound rather than a standalone library contribution claim.

These results describe this fixture, toolchain, and machine only. They are
retained as regression evidence, not release marketing claims.
