# Hardening audit: 2026-07-22

This report records the evidence-driven hardening review of `cli`. Source,
tests, generated API and reference artifacts, the pinned Cobra v1.10.2
adapter, workflows, dependencies, and documentation were reviewed together.
The framework owns the contracts below; parser-engine behavior is only
interoperability evidence.

## Reproducible baseline

Before behavior changes, `GOWORK=off make check` and
`GOWORK=off make release-check` passed. The initial mutation run killed every
executed mutant. All corrections were introduced with a failing regression,
then verified against the complete suite. The final release gate is the same
documented command used in CI.

## Public API and dependency boundary

The generated API baseline inventories every exported constant, variable,
type, function, generic parameter, option, interface, error, and method.
`TestPublicAPIMatchesBaseline` blocks accidental drift. Examples compile as
external-package consumers.

- Commands, handlers, middleware, bindings, renderers, completion providers,
  and shutdown controllers expose no Cobra type.
- The engine adapter is internal. A fresh mutable Cobra graph is built for
  each invocation; no mutable engine command escapes.
- Parser errors are translated into owned errors without third-party causes.
- Core has no package initializer, registry, environment read, working
  directory read, implicit file access, shell invocation, or `os.Exit`.
- Optional ecosystem composition is documented rather than imported; the
  runtime dependency boundary contains Cobra and its transitive `pflag` only.
- Constructors, zero values, nil and typed-nil callbacks, duplicate and
  ordered options, defaults, and custom value types have direct tests.

Compiled applications publish immutable owned definitions. No callback runs

lock or mutable parser state.

## Parser and conversion conformance

The black-box suite covers long, short, combined, assigned, repeated,
inherited, positional, remainder, and `--` syntax independently of Cobra.
It distinguishes omitted, defaulted, explicit empty, zero, and false values;
tests negative numbers, exponents, overflow, underflow, platform-sized
integers, durations, times, enums, paths, invalid UTF-8, Unicode, controls,
very large tokens, cumulative argv, and deep paths; and fuzzes argv and engine
translation separately.

| Boundary | Owned policy |
| --- | --- |
| Unknown or ambiguous input | sanitized owned usage error with bounded suggestions |
| Digit shorthand versus negative positional | command-local shorthand wins; otherwise numeric token remains positional |
| Values beginning with `-` | accepted through assigned syntax or where the binding owns the next token |
| Empty versus omitted | retained as distinct invocation value states |
| Enum/default declarations | validated at compile time; nonsecret values appear in metadata |
| Time values | declared layouts validated and published in generated contracts |
| Non-finite floats | follow `strconv.ParseFloat`; finite-domain policy belongs in validation |
| Unicode | byte-exact UTF-8 semantics; no normalization or confusable folding |
| Invalid UTF-8 and terminal/bidi controls | rejected at construction or sanitized in diagnostics |
| Windows paths | tokens are never shell-expanded, globbed, interpolated, or opened |

Comparison evidence is documented against
[standard `flag`](https://pkg.go.dev/flag),
[Cobra v1.10.2](https://pkg.go.dev/github.com/spf13/cobra@v1.10.2),
[urfave/cli](https://cli.urfave.org/v3/examples/flags/basics/),
[Kong v1.16.0](https://github.com/alecthomas/kong/tree/v1.16.0),
[Symfony Console](https://symfony.com/doc/current/components/console.html), and
[Laravel Artisan](https://laravel.com/docs/12.x/artisan). Divergence is an
owned policy unless the compatibility guide marks it unsupported.

## Lifecycle and failure matrix

The fixed lifecycle is parse, conversion, validation, middleware, handler,
middleware after-callbacks, cleanup, rendering, and exit selection.

| Failure or cancellation | Later work | Cleanup | Selected result |
| --- | --- | --- | --- |
| Parse/conversion/validation | handler not entered | registered cleanup once | primary owned error |
| Middleware before `Next` | downstream not entered | cleanup once | middleware error |
| Handler | after-callbacks unwind | cleanup once | handler error |
| After-callback | remaining unwind continues | cleanup once | joined without erasing primary |
| Cleanup | rendering still attempted | exactly once | joined after primary |
| Rendering/writer | no success after failure | already complete | output error joined/classified |
| Parent cancellation between phases | next phase not entered | cleanup once | cause-preserving cancellation |
| Panic | recovered only when explicitly configured | cleanup policy preserved | configured panic error |

`Next` is single-use and cannot resume after its middleware returns. Tests
cover every phase alone and in combinations, `errors.Is`, `errors.As`, joined
errors, cancellation causes, repeated runs, reentrancy, concurrency, and
short-circuiting.

## Output and exit compatibility

| Mode/class | stdout | stderr | Status |
| --- | --- | --- | --- |
| Human success | handler output | handler diagnostics | 0 |
| Human usage/validation | no success envelope | sanitized rendered error | 2 |
| Human operational failure | no success envelope | sanitized rendered error | 1 |
| JSON success | one `cli/v1` JSON envelope | empty | 0 |
| JSON failure | one valid deterministic JSON envelope | empty | classified |
| Quiet | empty | empty | classified |
| Cancellation/SIGTERM fixture | no success output | mode-specific | 130 |

JSON and quiet modes isolate direct handler stdout and stderr so application
text cannot corrupt machine output. Independent JSON decoding validates schema
and values. Human mode intentionally exposes application-owned streams.
Writers are exercised for short, zero, delayed, failed, closed, and broken
writes; cumulative output is bounded. The framework emits no ANSI color and
does not inspect TTY state or terminal width, making piped output deterministic.
Exit codes are range-checked and no library path exits the process.

## Help, completion, and generation

Help, Markdown, machine manifests, and Bash, Zsh, Fish, and PowerShell
completion are deterministic and graph-preserving. Generated references are
checked byte-for-byte. The owned shell templates pass Bash and Zsh syntax
checks, ShellCheck when available, and the PowerShell parser when available.
Fish is syntax-checked whenever `fish` is installed; it was unavailable on the
audit host.

Completion accepts token arrays, not shell strings. It bounds candidates and
descriptions, deduplicates deterministically, retains provider and writer
failures, honors cancellation, and redacts secret enums. Templates never use
`eval` or PowerShell expression evaluation. A live Bash attack fixture proves
candidate text cannot execute command substitution.

## Context, process, concurrency, and resources

Parent cancellation is checked before parsing and between every owned phase.
Explicit causes and deadlines survive classification. Shutdown subscriptions
are injected: delivery, repeated-signal policy, restoration, concurrent
controllers, idempotent close, and late-signal behavior are tested without
mutating process-global signal state.

The reference executable is the narrow process boundary. Subprocess tests
prove status 0 and 2 JSON behavior and SIGTERM-to-130 ECS-style termination.
The executable alone owns `signal.Notify`, `signal.Stop`, controller closure,
and `os.Exit`.

Race tests, twentyfold parallel/repeated execution, invocation-isolation
assertions, and cancellation/leak tests cover runners, output, completion,
generation, timers, contexts, channels, and subscriptions. Core starts no
goroutine per option and retains no invocation cache. An arbitrary application
`io.Reader`, `io.Writer`, handler, or provider can ignore cancellation; the
framework does not leak a goroutine attempting to preempt an interface it does
not own.

## Threat model and secret handling

Secret bindings are classified at construction and suppressed from help,
completion, manifests, defaults, errors, and suggestions. Diagnostics remove
ANSI CSI/OSC, terminal hyperlinks, bidi controls, carriage-return injection,
invalid UTF-8, and excessive content. Parser, edit-distance, formatting,
completion, generation, graph, argv, and output work is explicitly bounded.

The library cannot hide argv from process listings, shell history, CI metadata,
or orchestration APIs. Safer stdin, file, and provider composition is
documented. It also cannot redact application logs, telemetry, panic values,
human-mode direct IO, application globals, or a custom marshaler's output.
There is no shell invocation, unsafe expansion, implicit environment read, or
implicit file read. Dependency, vulnerability, license, secret, SBOM,
reproducibility, and workflow gates are mandatory.

## Non-interactive and integration evidence

Commands declare interaction capability. Noninteractive mode rejects required
interaction before handler side effects. CI, JSON, quiet, redirected, nil, and
non-TTY core execution never prompts or performs an implicit stdin read;
explicit application reads remain application-owned. Operations fixtures cover
ECS tasks, migrations, importers, backfills, diagnostics, and isolated parallel
runners. `prompts`, configuration, validation, logging, telemetry, and
correlation remain optional composition boundaries without dependency cycles.

## Documentation execution

README and documentation examples compile and execute as external consumers.
The required-document test enforces the API, adoption, cookbook, ECS, CI,
migration, import, backfill, diagnostic, completion, prompt-composition,
security, compatibility, limit, and release guidance. Generated docs and API
artifacts are drift-checked from a clean `GOWORK=off` module context.

## Findings registry

Owner for every resolved finding is the `cli` maintainers. No known finding
is deferred.

| Severity | Finding | Resolution |
| --- | --- | --- |
| High | Generated completion evaluated untrusted candidate text | Replaced engine templates with owned token-array protocols and attack tests |
| High | JSON output could be contaminated by handler IO | Isolated machine-mode streams and independently decoded envelopes |
| High | Lifecycle cancellation and late middleware continuation were inconsistent | Added phase boundaries, cause retention, and closed single-use continuations |
| High | Secret enum values leaked through generated metadata/completion | Redacted secret enums across all generated surfaces |
| Medium | Parser diagnostics exposed engine errors and terminal controls | Added owned boundary errors and public sanitization |
| Medium | Command grammars and option groups could compile ambiguously | Added graph ambiguity and satisfiability validation |
| Medium | Completion lost safe assigned/short values and provider failures | Completed owned syntax and joined protocol failures |
| Medium | Shutdown-derived contexts were not explicitly released | Made close idempotent and release deterministic |
| Medium | Metadata, output, suggestions, and candidate work lacked aggregate bounds | Added compile-time and runtime cumulative limits |
| Medium | Time/enum contracts were incompletely generated | Validated declarations and published nonsecret formats/values |
| Low | Compile options replaced rather than composed | Preserved deterministic option composition |
| Low | Digit shorthands conflicted with negative positional numbers | Made ambiguity command-local and tested both paths |
