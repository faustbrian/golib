# Security policy

Report suspected vulnerabilities privately through GitHub Security Advisories
for `faustbrian/identifier`. Do not include live identifiers, credentials,
customer data, or exploit traffic in a public issue.

Only the latest released minor version receives security fixes. Until the first
release, the `main` branch is the supported development line. A report should
include the affected family, input or generator configuration, impact,
reproduction, and whether entropy, clocks, persistence, or concurrency is
involved.
