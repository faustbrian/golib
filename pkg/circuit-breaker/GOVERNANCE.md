# Governance

Brian Faust is the initial maintainer and release authority. Maintainers merge
changes by technical evidence, API compatibility, security, and alignment with
the project boundary. Significant semantic changes should be proposed publicly
before implementation and require explicit release-note treatment.

Maintainers may be added after sustained, trusted contribution. Security reports
are handled privately; other decisions and conflicts are resolved in repository
issues or pull requests. Tagged releases require all gates in
[docs/verification.md](docs/verification.md) and an updated changelog.
