# Security policy

Do not include sensitive payloads or private barcode images in public reports.
Provide the affected package/version, minimized input when safe, expected and
actual resource bounds, and whether the issue can panic, hang, over-allocate,
or bypass validation.

Until a private publication contact is configured, report vulnerabilities to
the repository owner through GitHub's private vulnerability-reporting feature.
Decoded content is always untrusted and must not be executed or followed.
