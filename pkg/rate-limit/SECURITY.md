# Security policy

Report suspected vulnerabilities privately through GitHub Security Advisories
for this repository. Do not open a public issue with credentials, raw request
identities, exploit details, or production data.

The latest released minor version receives security fixes. Until v1.0.0, use
the latest commit selected by the project owner. Reports should include the
affected version, policy/backend, minimal reproduction, impact, and whether
trusted-proxy or key configuration is involved.
