# Roadmap

## Before v1.0.0

- Complete additional current-version ecosystem fixtures as independent
  implementations publish OpenRPC 1.4 documents.
- Maintain the established API compatibility, vulnerability, workflow,
  documentation, coverage, mutation, and integration gates.

## Optional after the core is proven

- Deterministic code generation that fails on unsupported schema semantics.
- A CLI for validate, format, bundle, diff, and inspect operations.
- Leaf adapters for owned query, validation, wire, HTTP-client, and localized
  packages where contracts can remain cycle-free.

Optional tooling will not delay fixes to core specification correctness or
weaken design-first support.
